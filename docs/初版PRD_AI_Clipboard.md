## 1. 产品概述

### 1.1 产品定位

一款专注于提升与 AI（Claude、ChatGPT 等）对话效率的轻量级剪贴板增强工具。

### 1.2 目标用户

- 频繁使用 AI 对话工具的开发者、创作者
- 希望在 AI 思考期间快速准备下一个 query 的用户
- 需要快速管理复制历史的 Windows 用户

### 1.3 核心价值主张

**"在 AI 思考的时候，你的思考也不被中断"**

- 毫秒级启动，快捷键一键唤起
- 无缝记录复制历史，随时回溯
- 极简交互，不打断心流

---

## 2. 用户痛点

| 痛点 | 描述 |
|------|------|
| **等待空白** | AI 生成回复时，用户想好了下一个问题，但没有地方记录 |
| **历史丢失** | 复制过的内容找不回来，需要重新查找 |
| **操作繁琐** | 打开笔记软件 → 新建文件 → 输入 → 复制，步骤太多 |
| **启动延迟** | 常规应用启动慢，打断思考节奏 |

## 3. 功能需求

### 3.1 功能优先级

| 优先级 | 功能模块 |
|--------|----------|
| P0 | 主输入框、快捷键打开、Enter 复制并关闭 |
| P0 | 剪贴板后台监控、历史记录（100条） |
| P0 | 历史记录弹窗、点击复制 |
| P1 | 历史记录搜索 |
| P2 | 窗口位置记忆 |

### 3.2 功能详情

#### 3.2.1 主输入窗口 (P0)

| 属性 | 规格 |
|------|------|
| 触发方式 | 全局快捷键 `Ctrl+Shift+Space` |
| 启动时间 | < 100ms |
| 界面形态 | 小型浮动窗口，仅包含输入框 |
| 窗口尺寸 | 约 600px × 120px（可配置） |
| 默认状态 | 打开后输入框自动获得焦点 |

**交互流程：**
```
用户按下 Ctrl+Shift+Space
    ↓
窗口弹出，输入框获得焦点
    ↓
用户输入内容
    ↓
用户按下 Enter
    ↓
内容复制到剪贴板 + 窗口关闭
```

#### 3.2.2 剪贴板后台监控 (P0)

| 属性 | 规格 |
|------|------|
| 监控范围 | 全系统文本复制操作 |
| 存储数量 | 最新 100 条 |
| 删除策略 | FIFO（先进先出），超出 100 条自动删除最旧记录 |
| 存储内容 | 复制时间 + 文本内容 |
| 数据持久化 | 否，程序关闭后清空 |

**技术要求：**
- 监控间隔 ≤ 500ms
- 内存占用 ≤ 50MB（常驻状态）
- CPU 占用 < 1%（空闲状态）

#### 3.2.3 历史记录管理 (P0)

| 属性 | 规格 |
|------|------|
| 触发方式 | 点击主窗口右侧小图标按钮 |
| 界面形态 | 弹窗/下拉面板 |
| 列表展示 | 每条显示前 20-30 字符预览 + 复制时间 |
| 鼠标悬停 | 显示完整内容（tooltip 或展开） |
| 点击操作 | 复制到剪贴板 + 弹窗自动关闭 |

**列表项 UI 设计：**
```
┌────────────────────────────────────┐
│ 📋 14:32  这是复制的文本内容的前三十... │
├────────────────────────────────────┤
│ 📋 14:30  def calculate_sum(items)... │
├────────────────────────────────────┤
│ 📋 14:28  https://example.com/api...  │
└────────────────────────────────────┘
```

#### 3.2.4 搜索功能 (P1)

| 属性 | 规格 |
|------|------|
| 触发位置 | 历史记录弹窗顶部搜索框 |
| 实时过滤 | 输入时即时更新列表 |
| 搜索范围 | 文本内容全量搜索 |
| 匹配方式 | 包含匹配（大小写不敏感） |

#### 3.2.5 窗口位置记忆 (P2)

| 属性 | 规格 |
|------|------|
| 存储内容 | 窗口最后关闭时的坐标 (x, y) |
| 恢复时机 | 下次打开时 |
| 边界检测 | 确保窗口在屏幕可见范围内 |

## 6. 技术方案建议

### 6.1 技术栈选型

| 组件 | 方案 A (Python) | 方案 B (Rust) |
|------|-----------------|---------------|
| 开发语言 | Python 3.11+ | Rust |
| GUI 框架 | Tkinter / PyQt6 | Tauri / egui |
| 剪贴板 | pyperclip | clipboard-win |
| 全局快捷键 | keyboard / pynput | global-hotkey |
| 打包发布 | PyInstaller | cargo + NSIS |
| 开发速度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 运行性能 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 资源占用 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**推荐方案：Python + Tkinter**
- 开发快速，易于迭代
- 满足性能需求（启动 < 100ms 可通过预热实现）
- 丰富的第三方库支持

## 9. 附录

### 9.1 参考产品

- **Ditto** - Windows 剪贴板管理器（参考历史记录 UI）
- **Paste** - macOS 剪贴板管理器（参考搜索交互）
- **Raycast** - macOS 效率工具（参考快捷键体验）

### 9.2 设计决策记录

| 决策 | 原因 |
|------|------|
| 不支持直接发送到 AI | 保持工具独立性，不依赖特定 AI 应用 |
| 数据不持久化 | 简化实现，历史记录多为临时查询用途 |
| 100 条上限 | 平衡内存占用与实用性 |
| Enter 复制并关闭 | 符合快速输入-发送的心流模式 |

### 9.3 术语表

| 术语 | 定义 |
|------|------|
| 剪贴板 | 系统级临时数据存储，用于复制粘贴操作 |
| 全局快捷键 | 无论当前焦点在哪个应用都能触发的快捷键 |
| FIFO | First In First Out，先进先出的数据队列策略 |
| 常驻后台 | 程序最小化到系统托盘，持续运行但不占用主窗口 |
