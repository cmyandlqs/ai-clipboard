## 1. 项目背景

### 1.1 产品定位

一款专注于提升与 AI（Claude、ChatGPT 等）对话效率的轻量级剪贴板增强工具。

**核心价值主张**：在 AI 思考的时候，你的思考也不被中断。

### 1.2 目标用户

| 用户群体 | 特征 | 核心需求 |
|----------|------|----------|
| **AI重度用户** | 每天使用ChatGPT/Claude等AI工具进行对话 | 等待AI回复时快速记录下一个问题 |
| **开发者** | 频繁复制代码片段、API链接等 | 快速管理复制历史，随时回溯 |
| **内容创作者** | 使用AI辅助写作、编辑 | 快速准备多个prompt，提高对话效率 |

### 1.3 市场验证

根据需求调研结果：
- ✅ AI等待时间是真实痛点（用户会同时开多个AI对话并行操作）
- ✅ 剪贴板管理工具市场成熟（Ditto、CopyQ、Paste等拥有大量用户）
- ✅ 轻量级工具更受青睐（Maccy <10MB、WeClipper 2.7MB）
- ⚠️ 市场上缺少"快速输入弹窗"这一差异化功能

### 1.4 差异化优势

| 竞品局限 | AI Clip 解决方案 |
|----------|------------------|
| 功能臃肿、UI复杂 | 极简专注：只有输入框+历史 |
| 启动较慢 | 常驻后台，快速唤起 |
| 快速输入需要多步操作 | Ctrl+Shift+Space → 输入 → Enter，三步完成 |
| AI功能与剪贴板分离 | 专为AI对话场景设计（后期可扩展AI集成） |

## 2. 用户画像

### 2.1 主要用户：小张（开发者）

```
姓名：小张
职业：后端工程师
工作场景：每天使用ChatGPT辅助编程
痛点：
  - AI生成代码时，想到新问题但没有地方记录
  - 复制过的代码片段找不回来
  - 打开记事本记录太麻烦，打断思路
期望：
  - 快速记录想到的问题
  - 随时找回之前复制的内容
  - 操作简单，不干扰工作流
```

### 2.2 典型使用场景

#### 场景一：AI思考时记录新问题
1. 小张向ChatGPT发送了一个代码问题
2. AI开始生成回复（需要等待5-10秒）
3. 等待期间，小张想到了下一个问题
4. 按下 `Ctrl+Shift+Space`，输入框弹出
5. 快速输入新问题，按Enter复制
6. AI回复完成后，直接Ctrl+V粘贴发送

#### 场景二：找回之前复制的代码
1. 小张记得10分钟前复制过一段代码
2. 按下 `Ctrl+Shift+Space` 打开主窗口
3. 点击历史按钮，浏览历史记录
4. 找到目标代码，点击复制
5. 切换到编辑器粘贴使用

## 3. 功能需求

### 3.1 功能优先级（MoSCoW）

| 优先级 | 功能模块 | 说明 |
|--------|----------|------|
| **Must Have** | 快速输入弹窗 | 核心差异化功能 |
| **Must Have** | 全局快捷键 | Ctrl+Shift+Space 唤起 |
| **Must Have** | Enter复制并关闭 | 符合快速输入心流 |
| **Must Have** | 剪贴板监控 | 后台监听复制操作 |
| **Must Have** | 历史记录弹窗 | 显示最近100条记录 |
| **Should Have** | 历史记录搜索 | 快速过滤查找 |
| **Should Have** | 窗口位置记忆 | 记住用户偏好位置 |
| **Could Have** | 数据持久化 | 重启后恢复历史 |
| **Could Have** | 快捷键自定义 | 允许用户修改 |
| **Won't Have** | 图片支持 | MVP阶段仅支持文本 |
| **Won't Have** | AI集成 | 后期版本考虑 |
| **Won't Have** | 云端同步 | 保持本地存储 |

### 3.2 功能详情

#### 3.2.1 快速输入弹窗（P0）

| 属性 | 规格 |
|------|------|
| 触发方式 | 全局快捷键 `Ctrl+Shift+Space` |
| 界面形态 | 小型浮动窗口，仅包含输入框 |
| 窗口尺寸 | 约 600px × 120px |
| 默认状态 | 打开后输入框自动获得焦点 |

**交互流程：**
```
用户按下 Ctrl+Shift+Space
    ↓
窗口弹出，输入框获得焦点
    ↓
用户输入内容
    ↓
用户按下 Enter
    ↓
内容复制到剪贴板 + 窗口关闭
```

#### 3.2.2 剪贴板监控（P0）

| 属性 | 规格 |
|------|------|
| 监控范围 | 全系统文本复制操作 |
| 存储数量 | 最新 100 条 |
| 删除策略 | FIFO，超出100条自动删除最旧记录 |
| 数据格式 | JSON（支持持久化） |

**数据结构：**
```json
{
  "id": "1701234567890",
  "content": "复制的文本内容",
  "timestamp": "2026-03-02T14:32:15",
  "preview": "复制的文本内容的前三十..."
}
```

#### 3.2.3 历史记录管理（P0）

| 属性 | 规格 |
|------|------|
| 触发方式 | 点击主窗口历史按钮 |
| 界面形态 | 弹窗/下拉面板 |
| 列表展示 | 每条显示预览 + 复制时间 |
| 点击操作 | 复制到剪贴板 + 弹窗关闭 |

#### 3.2.4 搜索功能（P1）

| 属性 | 规格 |
|------|------|
| 触发位置 | 历史记录弹窗顶部搜索框 |
| 实时过滤 | 输入时即时更新列表 |
| 匹配方式 | 包含匹配（大小写不敏感） |

#### 3.2.5 数据持久化（P1）

| 属性 | 规格 |
|------|------|
| 存储位置 | `%APPDATA%\AI Clip\history.json` |
| 保留时长 | 最近7天（可配置） |
| 清除方式 | 手动清除按钮 |

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 备注 |
|------|--------|------|
| 唤起时间 | < 500ms | 后台常驻后优化 |
| 剪贴板监控延迟 | < 500ms | 轮询间隔 |
| 搜索响应时间 | < 50ms | 100条数据 |
| 内存占用 | < 100MB | 常驻状态 |
| 启动时间 | < 2s | 首次启动 |

### 4.2 兼容性要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| 分辨率 | 最低 1366×768 |
| 多显示器 | 支持 |
| 字体 | 支持中英文 |

### 4.3 安全与隐私

- ✅ 所有数据本地存储，不上传云端
- ✅ 不收集用户使用数据
- ✅ 开源代码可审计
- ⚠️ 提示用户敏感信息（如密码）可能被记录

### 4.4 可用性要求

- 快捷键在任意应用中均可触发
- 支持键盘操作（无需鼠标）
- 界面简洁，学习成本低

## 8. 附录

### 8.1 竞品参考

| 产品 | 参考价值 |

| **Ditto** | 历史记录UI设计、快捷键交互 |

| **Maccy** | 键盘操作流程、轻量设计理念 |
| **Raycast** | 快速输入弹窗体验 |

### 8.2 设计决策记录

| 决策 | 原因 |
|------|------|
| 不支持图片（MVP） | 聚焦文本场景，简化实现 |
| 不直接发送到AI | 保持工具独立性，不依赖特定AI应用 |
| 数据支持持久化 | 用户需要回溯历史，提高实用性 |
| 100条历史上限 | 平衡内存占用与实用性 |
| Enter复制并关闭 | 符合快速输入-发送的心流模式 |
| 使用Tkinter而非PyQt | 内置无需安装，满足轻量需求 |

### 8.3 风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 全局快捷键冲突 | 无法唤起程序 | 允许用户自定义快捷键 |
| 剪贴板权限问题 | 某些应用可能阻止监听 | 提示用户授予权限 |
| Python打包体积 | EXE文件可能较大 | 使用UPX压缩 |
| 敏感信息泄露风险 | 密码等可能被记录 | 提示用户，提供清除功能 |

### 8.4 术语表

| 术语 | 定义 |
|------|------|
| 剪贴板 | 系统级临时数据存储，用于复制粘贴操作 |
| 全局快捷键 | 无论当前焦点在哪个应用都能触发的快捷键 |
| FIFO | First In First Out，先进先出的数据队列策略 |
| 常驻后台 | 程序最小化到系统托盘，持续运行但不占用主窗口 |
| 数据持久化 | 程序关闭后，数据保存到存储设备，重启可恢复 |
